# data/real — 実データ取り込み（後入れ・任意）

投資家スクリーニングの `データ待ち` / `推計` 項目を、**市区町村別の公式実数**へ差し替えるためのフォルダ。
CSV を置いて `python3 scripts/build_regions.py` を1回回すと `regions.json` にマージされ、
実データがある項目は自動で `実データ` 表示・スコアも実額ベースに切り替わる。無い項目は従来の推計にフォールバック。

## 主キー
`code` ＝ 国勢調査の市区町村コード `id_muni2020`（`source_pop_census.csv` と同一。例 札幌市=1100）。
各CSVは `code, pref, name` に加えて下記の値列を持つ。**全1,740行のコード・名称は雛形に投入済み**なので、値列を埋めるだけでよい（空欄＝未取込としてフォールバック）。数値はカンマ可・`NA`可。

## ファイルと列

### medical_facilities.csv — 厚労省 医療施設調査（静態）市区町村編
- `beds`     病床数（一般＋療養等の総病床）
- `zaishien` 在宅療養支援診療所（届出）数
- `houkan`   訪問看護ステーション数（介護サービス施設・事業所調査でも可）
- `hospital` 病院数（実数で置換したい場合のみ・任意）
- `clinic`   一般診療所数（同・任意）

### mui.csv — 厚労省 無医地区等調査
- `mui`    無医地区数
- `junmui` 準無医地区数
- `muiPop` 対象人口（任意）

### emergency.csv — 各都道府県 救急告示リスト（施設所在地で市区町村へ割付）
- `er2` 二次救急（救急告示）施設数
- `er3` 救命救急センター数

### industry.csv — 農林業／漁業センサス・観光統計
- `agri`    農業産出額（百万円）
- `fishery` 漁業産出額（百万円）
- `tourism` 延べ宿泊者数（千人泊）

## 反映されるスコア
- `beds/zaishien/houkan` → 供給空白カード＋病床充足率（S軸）
- `er2/er3` → 救急カード＋S軸に「救急空白」を加味
- `mui/junmui` → 無医地区カード＋無医地区リスク（有れば下限を引き上げ）
- `agri/fishery/tourism` → 産業実額カード＋素地スコアを実額ベースで算出（I軸）

すべて独自推計・仮説モデルの枠内。実額は出所を `regions.json` の `meta.sources` に併記のうえ運用すること。
